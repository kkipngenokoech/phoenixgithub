Agentic AI Class Project

PhoenixGitHub

A label-driven AI engineering system that turns GitHub issues into tested pull requests with human-in-the-loop review.

  • Pipeline: Plan → Implement → Test → PR
  • State machine via GitHub labels
  • Failure loop: ai:failed → Failure Analyst guidance → bounded relabel to ai:revise

Architecture

System Design

Trigger Layer

Watcher polls for ai:ready and ai:revise.

Orchestration

Coordinates full run lifecycle and retries.

Agent Layer

Planner, Coder, Tester, PR Agent, Failure Analyst + retry/feedback loop.

Tools Layer

src/phoenixgithub/tools: reusable git/path utilities.

Under The Hood: Decision Flow

Planner builds a file-targeted strategy, Coder applies edits, Tester validates output, then PR Agent prepares review-ready PR context.

Under The Hood: Memory + State

Watcher state, run metadata, issue comments, and label transitions persist context so retries are informed rather than stateless.

Under The Hood: Tool Usage

GitHub API operations, local git clone/branch/commit, validation command execution, and helper modules in src/phoenixgithub/tools.

Under The Hood: Failure Loop

Failure Analyst extracts root cause and actionable guidance; bounded auto-revise relabeling controls reruns and prevents infinite cycles.
